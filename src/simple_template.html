About IoT Anomaly Detection System

Neural Network Model

Our system uses a hybrid LSTM+RNN neural network architecture specifically designed for detecting anomalies in IoT network traffic:

  • LSTM Layers: Long Short-Term Memory layers capture temporal dependencies in network traffic patterns over time
  • RNN Layer: Recurrent Neural Network layer processes sequential data to identify unusual patterns
  • Quantization: Model is quantized to run efficiently on edge devices like Raspberry Pi
  • Sequence Analysis: Analyzes 60-packet sequences to detect deviations from normal traffic patterns
  • Anomaly Detection: Uses reconstruction error (MSE) to identify anomalous network behavior

How IoT Device Protection Works

IoT devices are particularly vulnerable to cyber attacks due to their limited security capabilities and constant connectivity. Our system protects IoT devices by:

  • Continuous Monitoring: Real-time analysis of network traffic to all connected IoT devices
  • Behavioral Analysis: Learning normal traffic patterns to detect deviations that could indicate attacks
  • Early Threat Detection: Identifying potential threats before they can cause damage
  • Automated Response: Generating alerts when suspicious activity is detected
  • Edge Computing: Processing data locally to reduce latency and maintain privacy

Common threats detected include DDoS attacks, unauthorized access attempts, unusual data exfiltration, and command injection attempts.

How the Program Works

  1. Network Discovery: Scans the local network to identify all connected devices using arp-scan
  2. Traffic Capture: Monitors network packets flowing to and from selected IoT devices
  3. Data Preprocessing: Converts raw network packets into features suitable for neural network analysis
  4. Pattern Recognition: The LSTM+RNN model analyzes 60-packet sequences to identify normal vs anomalous patterns
  5. Anomaly Detection: Compares current traffic patterns against learned normal behavior
  6. Alert Generation: Creates alerts when anomalies are detected with confidence scores
  7. Response Options: Provides notifications and potential automated responses to security events

The system is designed to run on a Raspberry Pi 5 with 8GB RAM, making it cost-effective and energy-efficient for continuous IoT security monitoring.
